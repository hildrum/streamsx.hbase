Operator HBASEPut

IBMStreams com.ibm.streamsx.hbase Toolkit > com.ibm.streamsx.hbase 2.0.0.commit_0f46c6e > com.ibm.streamsx.hbase > HBASEPut

Put tuples in HBASE, with support for checkAndPut. In the value is a primitive type, a Put must have a row, columnFamily, columnQualifier,and value specified. The row and value are from the input tuple, specified by the attribute rowAttrName and valueAttrName, respectively. The columnFamily and columnQualifier may be specified in the same way (via columnFamilyAttrName and columnQualifierAttrName respectively), or they may be the same for all tuples, by setting staticColumnFamily and staticColumnQualifier. Currently all must be of type rstring. If the value is a tuple type, then attribute names of the tuple will be interpreted as the columnQualifiers for the correponding values. See the PutRecord sample application for an example.To allow for locking, HBASE supports a conditional put. That is supported in this operator via the checkAttrName. If that parameter is set, then the input attribute it refers to must be a valid check type--see the description for details. On a put, the conditionis checked. If it passes, the put happens, if not, the put fails. To check the success or failure of the put, the operator has an optional output port. The attribute successAttr on the output port will be set to true if the put happens, and false otherwise.

Consistent Region

HBASEPut may be in a consistent region, but it may not be the start of a consistent region. At drain points, it flushes its internal buffer, and at resets, in clears its internal buffer.Thus, it ensures at-least-once tuple processing, but does not guarentee exactly-once tuple processing; thus if there is a reset, the same entry may be put twice. If used with HBASEGet to do a Get, modify, Put on the same entry in a consistent region, you could end up doingthe modification twice, so that is not recommended. If you need exactly-once tuple processing, using checkAndPut along with sequence numbers may be possible.

checkAttrName

Name of the attribute specifying the tuple to check for before applying the mutation. It must have a row, columnFamily, and columnQualifier. It may optionally have a value. When a value is specified, the put or delete operation will only succeed when that entry specified by the check attribute exists. When there is no value in the type of the checkAttribute, the put or delete operation will only succeed when there is no entry for that row, columnFamily, columnQualifer combination.
